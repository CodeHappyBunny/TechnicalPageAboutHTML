HTML Introduction

HTML is a markup language used to tell browsers how to structure webpages. It is made up of elements, which can be utilized to make content appear a certain way, as well as help arrange content within a document. You can also help style this content and give it flair with CSS, as well as further tweak the positioning of it.

Creating HTML Elements

First off, what is a HTML element made up of? Here are the following ingredients to make a simple HTML element that we need:

  • Ingredient 1: Opening Tag
  • Ingredient 2: Content
  • Ingredient 3: Closing Tag

How would we write that in code? Here is a example utilizing the paragraph tag to create a paragraph element:

<p>Spring is a great time to start gardening.</p>

Now as you can see the first part containing the < p > is the opening tag. The second part that is the sentence will appear on your document as text simply as:

Spring is a great time to start gardening.

The third part containing the < / p > is our closing tag.

Simple definitions to remember about the parts that make up an element:

  • Opening Tag--is the beginning of our element, the starting point to create it.
  • Content--is what will usually display on our document or it is what is enclosed within the element in this case the sentence within our paragraph element
  • Closing Tag--is the ending of our element, the completing point to creating it and without these tags it can create some bizarre results or even cause syntax errors.

Hopefully, this little bit of information can help you begin to create your own elements. In the next section we will talk about adding attributes, this should be fun!

Adding Attributes

We can give elements these cool things called attributes. You maybe asking yourself. what is a attribute? Well, an attribute contains information about a element, it won't appear in the content so don't worry about that, its part of the coolness of one.

What does an attribute need:

  • Definitely needs a space between it and the element name. When elements have more than one attribute they will need to be separated by spaces.
  • Of course most importantly, it will need the attribute name followed by a equal sign.
  • Lastly, it will need a attribute value which needs to be wrapped within quotation marks.

You maybe wondering how should an attribute being added to a element should look, well here we will demonstrate exactly just that utilizing our paragraph from the element demonstration:

<p class="springtime">Spring is a great time to start gardening.</p>

As you can see in red we added the attribute class which we can use to identify the element and target it for styling through CSS. In blue after the equal sign in quotations we have the attribute value which can be what ever you wish to classify the element as. Now that you have a basic understanding of this lets move on to talking about building HTML Documents a bit, you can find more attributes in the reference link below as well as more about them. Look at this small taste of attributes as the seasoning to your curosity to learn more about coding in HTML, as well as utilizing them in your code.

Building HTML Documents

All righty then! Let's dive into the nitty gritty of creating HTML Documents!!!! What's an HTML document you may ask yourself well its what we see when viewing a webpage in the window, it also gives it structure, basic format, and if you add a dash of CSS to it you can make it beautiful!(But we arent going to talk about CSS here you can find out more about it in the reference link below.)

We are going to utilize multiple elements to create a HTML document! The power of multiple steps coming together to build something amazing.Look at the lovely example below of all the cool code we typed to demostrate this:

<!DOCTYPE html>
 <html lang="en">
  <head>
   <meta charset="utf-8"/>
   <title>Spring Time</title>
  </head>
   <body>
    <p>Spring is a great time to start gardening.</p>
   </body>
</html>

So we just built a document titled "Spring Time" including the previous paragraph element we have been using. Okay, great now lets talk a little bit about the elements that make up the document as well as some of the attributes.


  • !DocType html--Basically it is something we add to make everything work, it's from the antiquity era of early HTML just make sure you include it when starting your build.
  • html--This is another super important element that some refer to as the "root element" you will wrap this element's tags around the content of your document following the DocType above.
  • head--This element is a container for all the things you want to include in your HTML page but that you don't want to show up or be a visible part of the content that is displayed. Things that can go in this element are keywords, page description that usually appears in the search results, CSS to style content, character declarations, and much more. We will talk more about this in the next section within "What Goes In The Head" section of this page.
  • meta charset="utf-8"--The meta element represents metadata and it cannot be represented by any other HTML meta elements such as base, title, link, script, or style. The charset sets up the character set for you UTF-8 is utilized most commonly and it includes most characters from majority of the written languages around the world.
  • title--This element creates a title for the page that displays in the browser tab where the page has loaded. It also is used as a description for the page when saving it to bookmarks.
  • body--This element contains all the content for your page that will be displayed. Things like for example, text, images, videos, games, playable audio tracks, and ect.

    • So now that we have learned a bit more information about the elements and their attributes that go into creating a HTML document let's continue on diving further by talking more about the head element next.

What Goes In The HEAD

Many important things go into the head of a HTML document.None of the awesome content within the head section displays in the web browser when it loads. The items within it are basically backstage working their magic. Anyways, some of the things contained with in a head element are title, links to CSS, links to custom favicons, and other metadata.

We have talked about the title element previously and it's use to create a title for the page that displays in the browser tab and also creates a name for the page when it is bookmarked. But let's talk about the difference between title and h1(header 1 element), some times these elements can be confused because they can be considered both the title of the page. But they are not the same the h1 element displays on top of the document within the browser and the title is only displayed on the browser tab. The h1 element is also located within the body element and not the head element.

Let's talk about the metadata element that is also located in the head, we briefly touched on it in the section above. Alot of different elements placed into head can be considered metadata and there are many different types of metadata but we are going to only bring up some of the more common ones. Previously we mentioned the charset attribute within a metadata element so we don't really need to touch on it again. Let's instead talk about some things within the meta element below:

  • name--it specifies what type of element it is and what type of information it contains.
  • content--specifies the acutal meta content

Adding these two things within the metaelement can help define who the author of the page is and the type of content on the page making it appear higher in the results for better search engine optimization. Here is an example of code of how we can utilize these two attributes within our meta element:

<meta name="author" content="Sonny Oak"/>
<meta
  name="description"
  content="Sonny, is Professor Oak's long lost son who is a professional gardener. He also, has collected many different plant type pokemon over the years."/>

As demonstrated above that is just a little bit about meta data. We aren't going to touch onto CSS but it can be linked into the head, if you want to utilize it to style your document.

Let's Learn A Little About BODY

Now, that we have a talked about the head, let's talk about the body, which is where our content that will be displayed in the document/web browser.The body contains paragraphs, headings, hyperlinks, tables, navigation, and ect. The body element can only be used once in a document.

The Div Element

The div element is just a generic container for the content within the body of the element. It is utilized for styling the content with CSS. It can be used to group content when adding a class or id attribute, or it can change the language of the section of a document when using the lang attribute.This element should only be used when the article element and the nav element are not appropriate.

Here's an example of a div element with a id attribute added:

<div id="springbox">
 <p>Spring is a great time for gardening.</p>
</div>

Utilizing the div element as you see here can allow you to set your selector to the id springbox and style it in CSS.

The Footer

The footer element usually goes towards the end of a document and it can contain a few different things let's list them here:

  • authorship information
  • copyright information
  • contact information
  • sitemap
  • back to top links
  • related documents

As you can see the footer can contain several different items, there can also be multiple footers within a document. Footers are located after the body element and within the html tags.
